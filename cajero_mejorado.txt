--------------------------------------------------------------
SITUACION PROBLEMATICA: CAJERO AUTOMATICO REALISTA
--------------------------------------------------------------

Un cliente quiere retirar dinero de un cajero automatico. El cajero solo entrega billetes
de $1000, $500 y $100 y el sistema debe entregar la menor cantidad posible de billetes para
cubrir el monto solicitado.
    El monto a retirar debe ser multiplo de 100 y no puede superar el saldo disponible
en la cuenta del cliente.

REQUISITOS ADICIONALES:
- El cajero debe simular la experiencia real de un cajero automatico
- Debe solicitar PIN de 4 digitos antes de permitir operaciones
- Debe mostrar un menu de operaciones disponibles
- Debe permitir consultar saldo y realizar extracciones
- Debe mantener un flujo continuo hasta que el usuario decida salir

---------------------------------------------------------------

ALGORITMO PRINCIPAL (CALCULO DE BILLETES):

1. Ingresar el monto de dinero a retirar por parte del cliente.
2. Evaluar si es multiplo de 100 y si no excede el saldo disponible del cliente.
    2.1 Si el monto excede el saldo disponible:
        Mostrar mensaje de que saldo insuficiente.
    2.2 Si no excede el saldo disponible pero no es multiplo de 100:
        Mostrar mensaje de que ingrese un monto redondo mayor o igual a $100
    2.3 Si cumple con ambas condiciones (else):
        Leer el monto ingresado y definir:
            2.3.1 Calcular billetes de $1000: monto // 1000
            2.3.2 Calcular restante: monto % 1000
            2.3.3 Calcular billetes de $500: restante // 500
            2.3.4 Calcular nuevo restante: restante % 500
            2.3.5 Calcular billetes de $100: restante // 100
            2.3.6 Mostrar cantidad de billetes de cada denominacion
            2.3.7 Actualizar saldo disponible

---------------------------------------------------------------

PSEUDOCODIGO DEL ALGORITMO PRINCIPAL:

INICIO
    LEER monto_a_retirar
    LEER saldo_disponible

    SI monto_a_retirar % 100 ≠ 0 ENTONCES
        MOSTRAR "El monto debe ser multiplo de 100."
    SINO
        SI monto_a_retirar > saldo_disponible ENTONCES
            MOSTRAR "Saldo insuficiente."
        SINO
            billetes_1000 ← monto_a_retirar // 1000
            restante ← monto_a_retirar % 1000

            billetes_500 ← restante // 500
            restante ← restante % 500

            billetes_100 ← restante // 100

            MOSTRAR "Billetes entregados:"
            SI billetes_1000 > 0 ENTONCES
                MOSTRAR " - De $1000: ", billetes_1000
            SI billetes_500 > 0 ENTONCES
                MOSTRAR " - De $500: ", billetes_500
            SI billetes_100 > 0 ENTONCES
                MOSTRAR " - De $100: ", billetes_100
            
            nuevo_saldo ← saldo_disponible - monto_a_retirar
            MOSTRAR "Monto retirado: $", monto_a_retirar
            MOSTRAR "Nuevo saldo: $", nuevo_saldo
        FIN SI
    FIN SI
FIN
